# 대한청소만세 CMS V1.5 설계서

## CMS 목적

대한청소만세 CMS V1.5는 대표가 코드 수정 없이 홈페이지 콘텐츠를 관리할 수 있도록 준비하는 관리자 화면입니다. 고객용 홈페이지와 관리자 CMS를 분리해 운영하며, 고객에게는 수정, 추가, 삭제 버튼이 노출되지 않는 구조를 전제로 합니다.

## V1.5 기능 범위

- `/cms/index.html`에서 PIN 6자리 로그인 제공
- 초기 PIN: `231204`
- 5회 실패 시 10분 잠금 구조
- 왼쪽 관리자 메뉴와 오른쪽 관리 화면 구성
- 홈페이지 미리보기 iframe 제공
- 작업사례, 고객후기, 현장기록, 비용안내, FAQ, 공지, 메인배너, 설정 메뉴 구성
- 샘플 JSON 데이터 불러오기
- 입력값을 브라우저 `localStorage`에 임시 저장
- 모바일 화면에서도 사용할 수 있는 반응형 레이아웃

## V2 예정 기능

- GitHub API 또는 별도 백엔드 연동을 통한 실제 JSON 파일 저장
- 관리자 PIN 변경 화면
- 이미지 업로드 파일 저장 및 압축
- 공지 예약 게시 자동 반영
- 고객 홈페이지 자동 빌드 및 배포
- 관리자 활동 로그
- 역할 기반 권한 분리

## 데이터 파일 구조

CMS V1.5는 고객 홈페이지 자동 반영을 염두에 두고 JSON 기반 데이터 파일을 준비합니다.

- `data/cases.json`: 작업사례
- `data/reviews.json`: 고객후기
- `data/journal.json`: 현장기록
- `data/prices.json`: 비용안내
- `data/faqs.json`: FAQ
- `data/notices.json`: 공지
- `data/banners.json`: 메인배너
- `data/settings.json`: CMS와 사이트 설정

## 고객 홈페이지와 연동 계획

V1.5에서는 기존 고객용 홈페이지 파일인 `index.html`, `css/style.css`, `js/script.js`를 변경하지 않습니다. CMS에서 다루는 데이터는 `/data` JSON 구조로 분리해두고, V2에서 고객용 홈페이지가 해당 JSON을 읽어 화면에 반영하도록 연결합니다.

관리자 CMS는 `/cms/` 경로에서 동작하도록 만들었으며, 추후 `cms.대한청소만세.kr` 같은 별도 관리자 도메인으로 확장할 수 있습니다.

## 테스트 방법

1. 로컬 서버를 저장소 루트에서 실행합니다.
2. 브라우저에서 `/cms/index.html`을 엽니다.
3. PIN `231204`로 로그인합니다.
4. 관리자 메뉴 전환이 되는지 확인합니다.
5. 작업사례, 고객후기, 비용안내, FAQ, 공지 입력 후 저장 동작을 확인합니다.
6. 샘플 데이터가 각 관리 화면에 표시되는지 확인합니다.
7. 홈페이지 미리보기 버튼으로 기존 고객용 홈페이지가 열리는지 확인합니다.
8. 브라우저 콘솔 오류가 없는지 확인합니다.

파일을 직접 열어도 기본 화면은 동작합니다. 다만 브라우저 보안 정책에 따라 JSON 파일 fetch가 제한될 수 있어, 정확한 샘플 JSON 로딩 테스트는 로컬 서버에서 확인하는 것을 권장합니다.
