# 대한청소만세 CMS V1.6 설계서

## CMS 목적

대한청소만세 CMS는 대표가 코드 수정 없이 홈페이지 콘텐츠를 관리할 수 있도록 준비하는 관리자 화면입니다. 고객용 홈페이지와 관리자 CMS를 분리해 운영하며, 고객에게는 수정, 추가, 삭제 버튼이 노출되지 않는 구조를 전제로 합니다.

## V1.6 기능 범위

- `/cms/index.html`에서 PIN 6자리 로그인 제공
- 초기 PIN: `231204`
- 5회 실패 시 10분 잠금 구조
- 왼쪽 관리자 메뉴와 오른쪽 관리 화면 구성
- 작업사례, 고객후기, FAQ, 비용안내 데이터 추가/수정/삭제
- 공지 데이터 추가/수정/삭제
- 작업사례 사진 드래그/선택 UI와 이미지 미리보기
- 브라우저 `localStorage` 기반 저장
- 새로고침 후 localStorage 데이터 유지
- 설정 메뉴에서 샘플 데이터 초기화
- 설정 메뉴에서 현재 데이터를 JSON 파일로 내보내기
- 모바일 430px 화면 대응

## localStorage 저장 방식

V1.6의 저장은 GitHub 자동 반영이 아닙니다. CMS에서 저장한 데이터는 브라우저의 `localStorage`에 `daehanCmsDraftData` 키로 저장됩니다.

따라서 같은 브라우저에서는 새로고침 후에도 데이터가 유지되지만, 다른 브라우저나 다른 기기에는 자동 공유되지 않습니다. 샘플 데이터로 초기화하면 localStorage 데이터가 삭제되고 `/data/*.json` 기준으로 다시 불러옵니다.

## 고객 홈페이지 반영 여부

V1.6에서 고객용 홈페이지에는 아직 자동 반영되지 않습니다. 기존 고객용 파일인 `index.html`, `css/style.css`, `js/script.js`는 변경하지 않고, CMS와 `/data` JSON 구조만 확장합니다.

고객 홈페이지 자동 반영은 V2에서 진행합니다.

## V2 예정 기능

- GitHub API 또는 별도 백엔드 연동을 통한 실제 JSON 파일 저장
- GitHub 자동 커밋 및 GitHub Pages 배포
- 고객 홈페이지에서 `/data/*.json`을 읽어 콘텐츠 자동 반영
- 관리자 PIN 변경 화면
- 이미지 업로드 파일 저장, 압축, 경로 관리
- 공지 팝업 또는 상단 배너 자동 게시
- 관리자 활동 로그
- 역할 기반 권한 분리

## 데이터 파일 구조

CMS는 고객 홈페이지 자동 반영을 염두에 두고 JSON 기반 데이터 파일을 준비합니다.

- `data/cases.json`: 작업사례
- `data/reviews.json`: 고객후기
- `data/journal.json`: 현장기록
- `data/prices.json`: 비용안내
- `data/faqs.json`: FAQ
- `data/notices.json`: 공지
- `data/banners.json`: 메인배너
- `data/settings.json`: CMS와 사이트 설정

## 고객 홈페이지와 연동 계획

V1.6에서는 CMS 저장 데이터를 localStorage에 보관하고, JSON 내보내기로 V2 연동 형식을 확인합니다. V2에서는 CMS 저장 버튼이 GitHub에 JSON 파일을 자동 커밋하고, 고객 홈페이지가 해당 JSON 파일을 읽어 화면에 반영하도록 연결합니다.

관리자 CMS는 `/cms/` 경로에서 동작하도록 만들었으며, 추후 `cms.대한청소만세.kr` 같은 별도 관리자 도메인으로 확장할 수 있습니다.

## 테스트 방법

1. 저장소 루트에서 로컬 서버를 실행합니다.
2. 브라우저에서 `/cms/index.html`을 엽니다.
3. PIN `231204`로 로그인합니다.
4. 고객후기 추가, 수정, 삭제 후 새로고침 유지 여부를 확인합니다.
5. 작업사례 추가, 이미지 미리보기, 수정, 삭제 후 새로고침 유지 여부를 확인합니다.
6. FAQ 추가, 수정, 삭제 후 새로고침 유지 여부를 확인합니다.
7. 비용안내 항목 수정 후 새로고침 유지 여부를 확인합니다.
8. 공지 추가, 수정, 삭제가 localStorage로 동작하는지 확인합니다.
9. 설정 메뉴에서 샘플 데이터 초기화와 JSON 내보내기를 확인합니다.
10. 430px 모바일 화면에서 가로 넘침이 없는지 확인합니다.
11. 브라우저 콘솔 오류가 없는지 확인합니다.

파일을 직접 열어도 기본 화면은 동작합니다. 다만 브라우저 보안 정책에 따라 JSON 파일 fetch가 제한될 수 있어, 정확한 샘플 JSON 로딩 테스트는 로컬 서버에서 확인하는 것을 권장합니다.
